# 考研英语单词 App — 模仿「不背单词」逐步实现计划

## 产品定位

基于「不背单词」核心框架，构建面向考研英语的 Web 端单词学习应用。核心理念：**真实语境输入 → 主动输出测试 → 艾宾浩斯间隔重复**。

### 1.2 数据模型设计（核心 6 张表）

```sql
-- 1. 用户表
users: id, username, email, password_hash, avatar, created_at

-- 2. 单词表（核心）
words: id, word, phonetic_us, phonetic_uk, word_audio_url,
       meanings(jsonb),           -- 多个释义 [{pos, def_cn, def_en, exam_weight}]
       root_affix(jsonb),         -- 词根词缀拆解 {root, affixes[], meaning}
       derivatives(jsonb),        -- 派生词 [word_id]
       frequency_rank,            -- 考频排名
       collocations(jsonb),       -- 词组搭配
       level                      -- 词库等级: cet4/cet6/postgraduate/ielts/toefl

-- 3. 例句表
examples: id, word_id, sentence, translation, source, audio_url, difficulty

-- 4. 学习记录表
learning_records: id, user_id, word_id,
  status (new/learning/review/mastered),
  ease_factor,           -- 艾宾浩斯算法参数
  interval,              -- 下次复习间隔（天）
  repetitions,           -- 复习次数
  last_review_at,        -- 上次复习时间
  next_review_at,        -- 下次复习时间
  correct_count,         -- 连续答对次数
  incorrect_count,       -- 累计答错次数
  created_at, updated_at

-- 5. 学习会话表（每日学习记录）
study_sessions: id, user_id, date, new_words_count, review_words_count, duration_seconds

-- 6. 错题/生词本
wordbook: id, user_id, word_id, type (wrong/favorite), created_at
```

## 第三阶段：艾宾浩斯智能复习系统

### 3.1 复习算法实现

基于 SM-2 算法（SuperMemo）改进：

```
算法参数：
- ease_factor (EF): 初始 2.5，范围 [1.3, 2.5]
- interval: 复习间隔（天）
- repetitions: 复习次数

答对时：
  if repetitions == 0: interval = 1 天
  elif repetitions == 1: interval = 3 天
  else: interval = interval * ease_factor
  ease_factor = max(1.3, ease_factor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)))

答错时：
  repetitions = 0
  interval = 1 天（24 小时后重新复习）
  ease_factor = max(1.3, ease_factor - 0.2)

quality 评分 (0-5):
  5 = 完美回忆，无需思考
  4 = 正确但犹豫
  3 = 正确但困难
  2 = 错误但看到答案后想起
  1 = 错误，完全不记得
  0 = 完全遗忘
```

- [ ] 实现 SM-2 算法核心函数
- [ ] 每次学习/测试后更新学习记录
- [ ] 计算 next_review_at 字段
- [ ] 后端：定时任务（每天凌晨）批量计算当日待复习单词
